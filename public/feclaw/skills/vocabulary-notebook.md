# Skill: 单词本

当用户遇到不认识的单词，或需要整理收集生词时，按以下步骤执行：

> **单词本文件位置**: `/workspace/vocabulary/vocabulary.md`（AI Agent 的工作区根目录下）
> 所有单词按日期追加到此文件中，用户可随时通过 Agent 查看或复习。

## 1. 建立单词本
- 执行: `mkdir -p /workspace/vocabulary`
- 如果 `/workspace/vocabulary/vocabulary.md` 不存在，创建它并写入初始标题: `# 📖 单词本`

## 2. 维护单词文件

### ⚠️ 信息收集原则
- **最大化利用用户提供的所有信息** — 用户发的文字、从图片中识别出的单词、上下文描述等，**全部都要记录**
- 如果是用户发来题目图片，从中提取生词一并收录
- 如果用户有额外说明（如「这个单词在阅读中见过」），也一并记录

### 文件格式（按日期分组）

每次添加时，先确认文件中是否已有今天的日期分区。如果没有，先插入日期标题。

```markdown
# 📖 单词本

## 2026-05-28

### abandon /əˈbændən/ v. 放弃
- **释义**: 放弃，遗弃
- **例句**: He abandoned his plan. — 他放弃了他的计划。
  - She never abandoned her dream. — 她从未放弃梦想。
- **来源**: 英语试卷阅读理解
- **注释**: 注意和 "abundant"（大量的）区分

### abduct /æbˈdʌkt/ v. 绑架
- **释义**: 绑架，诱拐
- **例句**: The child was abducted. — 那个孩子被绑架了。
- **来源**: 用户提到
- **注释**: 名词形式 abduction
```

### 2a. 添加生词

1. 读取 `/workspace/vocabulary/vocabulary.md` 看有没有今天的日期标题
2. 如果没有，在文件末尾追加 `\n\n## {今天日期}`
3. 按以下格式追加单词条目：
   ```
   ### {单词} /{音标}/ {词性}. {释义}
   - **例句**: {例句} — {翻译}
   - **来源**: {来源描述}
   - **注释**: {个人备注}
   ```
4. 如果用户提供了多个信息（多个例句、详细用法），全部记录

### 2b. 用户发图片
如果用户以图片形式发送包含生词的内容：
1. 使用 `file_read` 查看当前单词本
2. 使用 spawn_subagent 读取图片，提取其中的单词
3. 按上述格式追加到单词本中

### 2c. 用户发文字
如果用户直接说出/输入不认识的单词：
1. 读取当前单词本，确认是否已有此词（避免重复）
2. 如果已有就告知用户；如果没有，创建条目
3. 用 `file_write` 或 `file_append` 将新条目写入文件

## 3. 整理后
- 告知用户已收录
- 可主动询问是否需要补充音标、例句或用法
- 可询问是否还有更多生词需要添加
