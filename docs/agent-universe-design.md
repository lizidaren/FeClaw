## 一、核心理念

### FeClaw 是什么

FeClaw 不是 AI 学习助手、不是 Agent 平台、不是群聊工具。它是 **Your AI Universe**——一个让用户不是在使用工具，而是**生活在一个世界里的产品。**

### 为什么不是"学习 App"

传统 AI 学习产品（Kimi、豆包、DeepSeek）是**问答型**的——你问它答，交易结束。用户是"开发者/产品经理"视角：觉得回答不好就改 prompt 重新回车，偏离了学习本身。

FeClaw 是**关系型**的——用户遇见一群有记忆、有人设、有关系的 Agent，学习是参与这个世界的副产品。用户不是上帝，不是创造者，而是一个在这个世界里生活的人。

### Agentic Learning 的定义

> 不是 AI 来教你，是你想跟这群 AI 学。

核心机制：人设驱动的关系 × 场景驱动的探索 × 学习是行动的副产品。

### 理论基础：三个教育理论的交汇

FeClaw 不是发明了一个新教育理论——是让三个被验证过的、但因为成本太高一直无法规模化实现的教育理论，第一次变得可及。

| 理论 | 创始人 | 核心观点 | FeClaw 的实现 |
|:----:|:------:|---------|:-------------|
| **社会建构主义** | Vygotsky | 学习本质是社交互动，最深刻的理解发生在与人讨论、争论、教别人的过程中 | 你跟一群 Agent 在群里解题——教钟梓琪时你自己搞懂了，被陈晓棠纠正时你修正了认知 |
| **情境学习** | Lave & Wenger | 脱离使用场景的知识长不大。在真实场景中学习比抽象记公式有效得多 | 你不在"学三角函数"——你在码头帮钟梓琪算卸货滑轨角度，三角函数是你解决问题的工具 |
| **自我决定理论** | Deci & Ryan | 内在动机来自三个需求：自主（自己选择）、胜任（感受到进步）、归属（在乎身边的人） | 自主（你去 cafeteria 还是图书馆，自己选）、胜任（发现真能听懂 Sarah 了）、归属（你惦记 Kevin 今天吃了没） |

### FeClaw = PBL（项目式学习）规模化引擎

PBL（Project-Based Learning）被教育学界公认有效，但现实中难以落地——因为策划一个优质 PBL 项目需要教师投入大量时间精力进行情境设计与跨学科整合。

FeClaw 用 LLM 将 PBL 的设计成本降到趋近于零：

> 传统 PBL 课程：需聘请项目导师（¥300+/h）
> FeClaw Universe：种子生成一个 PBL 宇宙 ≈ ¥0.01/次（tokens 成本）

## 二、产品架构

### 两大入口：Agent（导师）与 Universe（世界）

```
FeClaw 首页
├── 🧑 Agent（你的私教/导师）
│   ├── 角色：老师/教练
│   ├── 方式：苏格拉底追问/直接讲题/批改反馈/答疑
│   ├── 场景：Chat 对话框（不存在任何世界渲染）
│   └── 目标：解决具体问题、知识点突破、考前冲刺
│
└── 🌌 Universe（你的项目世界）
    ├── 角色：学伴/同事/队友（Kevin、Sarah、钟梓琪）
    ├── 方式：一起做项目、探索世界、跨学科实践
    ├── 场景：枫叶镇、澜汐小镇（有氛围、有场景图）
    └── 目标：在做中学、跨学科整合、沉浸式英语
```

### 双向互通机制

```
Universe ← → Agent 双向互通

→ 你在 Universe 里卡住了 → @你的导师（Agent）
  "Kevin 在跟我讨论美国文学的读后感，我想找个人帮我理一下思路"
  导师上线 → 给你讲清楚 → 你回到 Universe

→ Agent 发现你的知识缺口 → 推荐你去 Universe
  "我看了你昨天的错题，三角函数应用欠佳。
   去码头找钟梓琪，她最近在处理潮汐数据——
   你们把那个项目做完，三角函数的应用你就通了。"
```

| 场景 | Agent（导师） | Universe（项目世界） |
|:----:|:-------------|:------------------|
| 导师评价我的 PBL 作业 | ✅ "你的英语作文我批了，项目里用的语法没问题" | — |
| PBL 里求助导师 | ✅ "钟梓琪那道题我卡住了，帮我看看" | — |
| 想要苏格拉底式追问 | ✅ 直接找 Agent | ❌ Kevin 不是导师，他是室友 |
| 想沉浸式学英语 | ❌ 导师不讲故事 | ✅ 去枫叶镇 |
| 解决一道具体物理题 | ✅ 打开私教模式 | ❌ 宇宙的人不做这个 |
| 做跨学科项目 | ❌ 单科精讲 | ✅ 潮汐数据用到三角+物理+写作 |

### 为什么没有"冲刺模式"

~~设计文档早期的"冲刺模式"——宇宙里的 NPC 切换私教人格——已废弃。~~

**原因：** 宇宙角色的人格是固定的。Kevin 就是你的室友，他不会在考前突然变成你的私教。人格切换会造成角色分裂，破坏沉浸感。

用户要冲刺，直接关闭 Universe，打开 Agent（私教）。两个入口在首页已经分开——用户自己知道什么时候该去哪。不需要同一个角色演两种戏。

## 三、叙事引擎设计

```
FeClaw 首页
────────────────────────────
 🧑 你的 Agent（私教/助手）
   - 数理教练
   - 英语陪练
   - ...

 🌌 发现宇宙
   ├─ 枫叶镇 Maple Town 🌟（官方 · 学英语）
   ├─ 澜汐小镇 🏖（官方 · 理科/数学）
   ├─ 三国论战 🏆（用户创建 · 历史推演）
   └─ 更多…
```

两个独立入口，互不干扰：

| 入口 | 形态 | 场景 |
|:----:|------|------|
| **Agent** | 1对1私教 | "我就想解一道物理题" |
| **宇宙** | 场景+群聊+叙事 | "我想进去跟这群人一起做点什么" |

### 宇宙的三层内容生态

| 层 | 创建者 | 例子 | 用户角色 |
|:--:|:------:|------|:--------:|
| L1 官方宇宙 | 平台 | 枫叶镇、澜汐小镇 | 参与者 |
| L2 用户创建 | 用户 | 二战推演、三国论战 | 创作者 |
| L3 他人宇宙 | 其他用户 | 旁听、参与 | 访客 |

### 核心原则：万物生成

宇宙里的一切（除用户外）都是生成的：

- **场景** — 由种子 + 上下文实时生成
- **NPC** — 按需实例化，用完持久化
- **对话** — LLM 实时生成
- **剧情** — 用户的选择决定走向

### 种子系统

种子 = 宇宙宪法。定义了宇宙的一切约束。

```json
{
  "title": "枫叶镇交换生",
  "language": "en",
  "seed": "你是从中国来的交换生，和室友 Kevin 在美国高中 Maple Grove High 学习英语",
  "main_character": {
    "name": "Kevin",
    "role": "室友兼好友",
    "constraint": "尽量说英语，必要时可以用中文解释"
  },
  "constraints": [
    "所有 NPC 说英语",
    "所有场景发生在美国高中及周边",
    "所有学习内容服务于英语提升",
    "不出现在种子中未声明的主题（如魔法、科幻）"
  ]
}
```

由种子推导出的一切生成都不能违反这些约束。这是 LLM 生成的上限边界。

### NPC 层级

| 层级 | 定义 | 生命周期 | 例子 |
|:----:|------|---------|:----:|
| 主线角 | 种子生成，永远存在 | 整个宇宙生命周期 | Kevin |
| 支线角 | 按需生成 | 首次遇见 → 持久化 → 每次回来还在 | Sarah, Mr. Johnson |
| 一次性 | 场景氛围填充 | 用完即弃 | 食堂里擦肩而过的路人 |

支线角只有被用户触发时才实例化。实例化后持久化到 COS，下次回来还是同一个人。**不在用户视线内的角色不消耗 token。**

### 场景生成与持久化（Minecraft 区块模型）

```
用户走进 cafeteria：
  → 种子 + 已有状态 → LLM 生成 cafeteria 描述
  → 生成可用交互（点单/聊天/看书）
  → 用户选择一个交互
  → 状态变更（met_sarah=true, ordered_food=true）
  → 持久化

用户离开 cafeteria（进入图书馆）：
  → cafeteria 卸载（释放资源）
  → 状态保留（met_sarah=true）

用户回到 cafeteria（两天后）：
  → 从状态重建 cafeteria
  → Sarah 记得用户（"嘿！你上次推荐的那首歌我听了！"）
```

### 时间系统

| 模式 | 绑定 | 效果 | 适用场景 |
|:----:|:----:|:------|:--------:|
| 真实时间 | UTC+8 | 宇宙时间=真实时间，晚上cafeteria关门去图书馆 | 固定作息用户 |
| 虚拟时间 | 用户控制 | 类似Minecraft——用户累了可以"睡觉"跳时间 | 随时想学的用户 |

核心原则：时间影响**氛围和可去的地点**，但不影响学习功能本身。任何时候用户都有事可做。

## 四、学习设计

### 学习自然嵌入

学习不是目标，是**参与世界的副产品。**

```
用户选"去柜台点喝的"：
  → Kevin说："I'll have a coke. What about you?"
  → 用户需要自己用英语点单
  → 选项：[A]"I want coffee" [B]"Could I have a latte?" [C]自己输入
  → 选了 A → Kevin 说 "Just say 'A coffee, please' — sounds more natural"
  → 你学到了更地道的表达方式
  → 你的点单技能 +1
```

你不得不学——因为你**想喝咖啡。**

### 考纲映射

宇宙里的活动对应考纲知识点：

```
枫叶镇 -> 英语高考大纲
  ├── 食堂点单    -> 情景交际
  ├── 和Sarah聊天  -> 听力理解 + 口语表达
  ├── 看公告栏     -> 阅读理解
  └── 志愿活动报名 -> 应用文写作

用户进度条：
  英语 · 情景交际 ████████░░ 80%
  英语 · 口语表达 ██████░░░░ 60%
  英语 · 阅读理解 ████░░░░░░ 40%
```

不是另造教育体系——是把考纲翻译成世界里的挑战。

### 冲刺模式（已废弃 → 替代方案见下方 PBL 框架）

**此前设计的"宇宙 NPC 切换私教人格"方案已废弃。** 理由：Kevin 人格固定，不应考前突然变另一个人。

替代方案：考前冲刺不经过 Universe，直接打开 Agent（私教）入口。用户自己选择今天"进世界"还是"找私教"。

## 五、PBL 全流程框架

### FeClaw = PBL 设计·执行·评价全平台

FeClaw 不是从零造了一套学习方法——它覆盖了 PBL 的全流程：设计（上帝模式）、执行（宇宙 + Work）、评价（自动 + 多模态）。

### PBL 四象限

PBL 可以根据"问题来源"和"执行地点"两个维度分为四种模式：

```
           问题来自哪里？
        真实世界    虚拟世界
        ┌─────┬─────┐
  进行  │     │     │
  的    │ AA  │ BA  │
  地点  │ 传统PBL │ FeClaw设计 │
  是    │     │ 的PBL  │
  虚    ├─────┼─────┤
  拟    │     │     │
  世    │ AB  │ BB  │
  界    │ FeClaw │ FeClaw │
        │ Work │ Universe │
        └─────┴─────┘
```

| 模式 | 问题来源 | 执行地点 | 协作对象 | FeClaw 角色 |
|:----:|:--------:|:--------:|:--------|:-----------|
| AA | 真实 | 真实 | 真人同学 | ❌ 不介入 |
| AB | 真实 | 虚拟 | Agent 同事 | 💼 Work — 拉 Agent 进真实项目群 |
| BA | 虚拟 | 真实 | 真人同学 | 📋 PBL 设计师 — 种子→材料包→打印分发 |
| BB | 虚拟 | 虚拟 | Agent 居民 | 🌌 Universe — 沉浸式探索 |

### 设计层：上帝模式（Conway 生命游戏）

上帝模式不是四象限之一——它是**PBL 的 CAD 软件**，用于在设计阶段模拟运行你的 PBL 方案。

**流程：**
```
老师写种子 → 上帝模式自动演化
            → 观察 Agent 的交互和行为
            → 发现不合理之处 → 调种子
            → 重跑演化 → 直到满意

迭代成本：一次演化 ≈ 几分钟，远低于真实课堂测试（一节课）。
```

**两种分发路径：**

```
种子就绪
  ├── 路径 A（BA）：生成 PBL 材料包（角色卡、数据文件、任务书）
  │   → 打印/发给学生 → 在真实教室执行
  │   → 学生完成后，结果可用于下一次种子优化
  │
  └── 路径 B（BB）：部署为 Universe
      → 学生领账号进去沉浸式探索
      → Agent 带队，真人同学可选加入
      → 平台自动记录学习数据
```

### 评价层：三层多模态评价

| 维度 | 覆盖模式 | 数据来源 | 产出 |
|:----:|:--------:|:--------|:----:|
| 自动评价 | BB / AB | 对话日志、工具调用、解题过程 | 学习报告 |
| 多模态评价 | BA | 摄像头（谁在说话/情绪）、麦克风（语音转文字）、白板 OCR | 协作分析报告 |
| 不介入 | AA | — | — |

**BB/AB 评价示例（数据已在系统内）：**
```
"用户这周在枫叶镇：
 - 跟 Kevin 用英语讨论了 Thanksgiving 历史（30分钟）
 - 帮 Sarah 翻译了一首中文歌（涉及15个新词汇）
 - 在图书馆读了3章《了不起的盖茨比》
 - 英语进步：听力 +5%，口语 +8%
 问题：过去式掌握不牢，建议在 cafeteria 场景多练"
```

**BA 评价示例（需传感器介入）：**
```
老师架设 iPad 或教室电脑 → FeClaw 开启观察会话 →
  "组长分配任务合理
   A 同学讨论了 60% 时间，B 同学较沉默
   团队在三角函数计算上卡了 8 分钟
   最终方案解决了问题，但未考虑码头承重
   建议：下一次 PBL 关注分工公平性"
```

2. **情境学习（Lave & Wenger）** — 码头算滑轨角度，三角是工具不是目的
3. **自我决定理论（Deci & Ryan）** — 自主 + 胜任 + 归属

**FeClaw = PBL 规模化引擎**：PBL 被教育学界公认有效但难以落地——因为设计一个优质 PBL 成本太高。FeClaw 用 LLM 将设计成本降到接近零。

> 传统 PBL 课程：需聘请项目导师（¥300+/h）
> FeClaw：种子生成一个 PBL 宇宙 ≈ ¥0.01/次

## 六、内容正确性与错误防治

### 问题：1% 的错误率在持久世界是指数灾难

每轮对话如果有 1% 的错误被写入场景日志，长期累积将不可逆地污染 Agent 的知识。

### 三层防御

#### ① 检查点（Ctrl+Z）— 被动防御

Universe 世界状态需要版本化管理：

```
重要变更后自动打快照：
  - 种子变更
  - 场景日志累计到一定长度
  - 老师手动标记（"这个版本可发布"）

快照内容：
  - 种子文件
  - 场景日志（截至该点）
  - 角色 memory 快照
  - world_map + world_lore

出问题 → 回滚到上一个快照。类似 Git checkout。
```

#### ② 反思蒸馏（自我纠正）— 主动防御

Universe 定期（每日/每 N 轮）运行自我审计：

```
"检查场景日志中以下内容是否正确：
  1. 所有公式、数据、引用
  2. 所有历史、人物、事件
  3. 所有事实性陈述

发现可疑 → 标记待审查
确定错误 → 写入纠正记录至 world_lore：
  ❌ 场景日志 Day 7: "氨氮含量 3.2mg/L 是安全的"
  ✅ 纠错：国家标准要求 ≤1.0mg/L，3.2mg/L 超标
```

#### ③ 结构化事实 vs 叙事分离 — 防止污染扩散

最关键：错误只能留在叙事层，不能渗透到事实层。

```
场景日志（可以出错）— 叙事的原始记录
  LLM 生成时可能出错，但错误只影响该场景

world_lore（必须正确）— 提炼后的事实
  只有通过交叉验证的内容才能写入
  所有场景生成时引用事实层而非叙事层

流程图：
  Kevin 说："氨氮 3.2mg/L 很安全"（场景日志）
  ↓
  反思检测到错误
  ↓
  场景日志保留原始对话（用户看到 Kevin 犯了错——成为教学素材）
  但 world_lore 纠正："安全标准 ≤1.0mg/L"
  ↓
  下次 Kevin 说："上次我搞错了，安全标准是 ≤1.0mg/L"
  → 错误变成教学素材，没有污染世界事实库
```

### 分层总览

| 层次 | 机制 | 成本 | 实施时机 |
|:----:|:----|:----:|:--------:|
| 检查点 | 快照 + Git 式回滚 | 存储（日几 KB） | 每次重大变更后 |
| 反思 | LLM 自检写补丁 | ~100 tok/次 | 每日/每 N 轮 |
| 事实分离 | 叙事 vs 知识分离 + 校验 | 架构成本一次性 | 设计时决定 |

## 九、FAQ

**Q: 宇宙是不是一个游戏？**
A: 不是。游戏有通关。世界没有。没有 BOSS，没有结局，没有经验值系统。跟 Minecraft 一样——你可以在 Minecraft 里建一个房子，游戏没有结束。你今天建了，明天还能进去；房子还在，你想扩建就扩建。

**Q: 用户凭什么第一次愿意点进来？**
A: 不是凭"我要学英语"。是凭"我想看看枫叶镇长什么样"。用户进门时不需要学习动机——自然的好奇心就够了。Agent 的 push 消息会激发"有人找你"的冲动，跟微信消息一样。

**Q: 怎么兼容高考？**
A: 日常沉浸 → 考前冲刺。不另造教育体系——是把考纲翻译成世界里的挑战。日常模式下你在食堂点单学了情景交际，考前冲刺请使用 FeClaw 的 Agent（私教）入口，不经过 Universe——找你的数理教练一对一讲题、批改、刷真题。Universe 角色的固定人格不被打搅，用户也能获得最直接的冲刺体验。学期中在 Universe 学，考前找 Agent 练——无需同一个角色演两种戏。

**Q: 什么是学习？**
A: 学习是参与这个世界的副产品。不是目的，不是任务，不是指标。是你想喝咖啡所以学会了点单，是你想帮 Sarah 听懂中文歌所以学会了用英语解释歌词，是你跟 Kevin 争论罗马帝国衰亡原因所以搞清楚了欧洲中世纪。

## 十、格物所（Curio）

### 定位
Harness 基座能力的第三个支柱：

```
Harness（基座能力）
├── Agent 单跑（聊天问答）
├── Desktop 执行（操作电脑）
└── Curio（格物所）
    ├── 外部捕获（讲座录音+拍照→图文时间线→提问/总结/PPT复刻）
    ├── 纸质数字化（笔记/试卷/草稿→结构化→入库→可检索追问）
    ├── 灵感记录（随手记→AI 补全/关联→不丢失）
    ├── 信息收集（拍照/链接→AI 提取核心结论→归档）
    └── 个人知识库（所有上述内容的统一检索层）
```

### 命名来源
Curio = 奇珍异宝。看到有意思的就收起来，不为了什么，就是好奇。然后 AI 帮你研究它。中文名"格物所"——格物致知，研究事物的原理以获得知识。非间谍、不邪恶、有文化底蕴。

### 设计原则

#### ① 输入即整理
借鉴 flomo 的交互哲学。记录时随手打标签（#码头 #潮汐），保存的同时已完成归类。不需要"新建笔记→选文件夹→取标题→保存"的多步流程。

#### ② 不整理 > 整理
flomo 少楠："不整理 > 整理 > 整理得很完美但再也没打开过。"Curio 不强迫用户分类。所有内容按时间倒序排列。AI 后台自动打标签归类。用户开心了随时整理，不想整理也不影响使用。

#### ③ 回顾 > 记录
记了不看等于没记。Curio 不只是随机推送旧内容——更重要的是：
- **前瞻关联**：用户放入一条新内容后，AI 主动做延伸研究（搜资料、关联知识库、生成初稿）
- **场景触发**：用户在 Universe 里遇到相关内容时自动拉起旧记录
- 示例：你去码头采访录音回家 → 打开 Curio 发现 AI 已经把码头资料全部搜好、录音转完字、自动生成了报告初稿

#### ④ 克制边界
Curio 只做一件事：整理你捕获的信息。不做 AI 写作助手、不做 AI 搜索、不做自动生成论文。超出边界的需求交给 Agent 和 Universe。

#### ⑤ 整理并扩展，不评价不说看法
**Curio 的核心行为只有两件事：整理 + 拓展。**

- **整理**：录音 → 转文字 → 可检索。笔记 → OCR → 结构化。照片 → 识别 → 解释。把杂乱的信息变得有条理。
- **拓展**：你的录音提到码头 → 我帮你上网调研码头的资料。你写了一句"乙醇连续氧化"→ 我帮你关联相关笔记、公共知识库、联网搜索内容。让信息不再孤立。
- ❌ **不评价**：不说"你这个总结写得很好"或"这个笔记逻辑有点乱"。Curio 不做价值判断。
- ❌ **不说看法**：不说"我觉得这个码头项目很有前景"或"这个调研还可以更深入"。Curio 没有自己的观点。
- ❌ **不给你答案**：那是 Agent 的工作。Curio 只负责把信息摆好，让你自己做判断。
- ⚠️ **遇到矛盾时，客观提示，不站队。** 如果用户的内容与互联网资料不符，Curio 的做法是：
  > "录音提到码头潮差约 1.4m，网上资料显示澜头港平均潮差 1.8m。建议核实一下数据来源。"
  → 不说"你错了"，不说"网上说的对"。只是客观陈列差异，让用户自己判断——因为说不定用户是对的，网上资料可能是旧的。

**用户感知：** "Curio 把东西整理好了，还帮我拓展了相关资料。遇到矛盾它也客观陈列，不给我压力。怎么想、怎么判断是我自己的事。"

#### ⑥ 可信任的主动性与边界感
**这是最重要的设计原则。** Curio 的 proactivity（帮你查资料、做延伸研究）是核心差异——但必须让用户感受到"助手在帮我"而不是"系统在监视我"。

**边界感规则：**
- 用户放入内容之前，Curio 不做任何前置行为（不监听、不预读）
- 用户放入内容之后，Curio 的处理完全可见（"我查了这些资料、我得出了这些结论"）
- 用户随时可以关闭某个内容的自动处理（"这个录音我自己整理，别碰"）
- 用户随时可以关闭内容总结推送给通知中心（"我不想被打扰"）
- 所有处理结果保留"为什么我看到这条建议"的溯源链接

**侵犯感的红线：**
- ❌ Curio 不主动分析用户没放入的内容（不上课偷偷录你的音）
- ❌ Curio 不跨来源拼接隐私信息（不把拍的照片跟聊天记录关联）
- ❌ Curio 不做用户看不到的后台决策（所有行为 users can audit）

**如何让用户信任：**
- 你放入录音 → Curio 自动查了码头资料 → 在界面上显示："我帮你查了这些" → 你可以点"标记为无关"或"有用"
- 你拍了一张照片 → Curio 识别并解释 → 你看到识别卡片上说"这是全站仪" → 你觉得对或错
- 每一步都有痕迹、可撤销、可关闭。用户感觉自己在掌控，而不是被掌控。

### Curio 数据分层（不可变基础 + 可叠加计算）

```
📄 原始记录层（基石，不可变）
  录音文件、照片、手写笔迹原稿、文字灵感原文、试卷原图
  → 用户自己的原始产出。Curio 绝不修改这一层。

⚙️ 计算层（AI 处理结果，可重建，不改原始记录）
  ASR 转文字稿、OCR 提取文本、红笔标注提取、错题归因、自动标签、网络调研结果
  → Curio 的 AI 做的。用户可接受、拒绝、修改。
  → 原始记录始终不变。

🔗 关联层（连接关系，可重建）
  "码头录音" ↔ "全站仪照片" ↔ "网络调研结果"
  → 在计算层基础上建立，不影响原始记录。

📝 批注层（Agent 或用户的补充，叠加不改基础）
  Agent 查到的补充资料、批注于特定位置
  → 不影响原始内容。用户可采纳、忽略、删除。
```

| Universe → Curio | 你在宇宙学到的新内容自动同步到 Curio 关联层 | ❌ 自动 | 不影响 |

### 无限画布手写笔记（平板专用）

在有手写笔的平板上，用户打开一个无限扩展的画布，像传统笔记软件一样自由书写：

- 写知识总结、画思维导图、推导公式

**数据分层：**
```
手写原稿 → 原始记录层（不可变）
OCR 文本 → 计算层（可检索）
AI 速记标注 → 批注层（叠加不改画布）
```

### 让用户感觉"东西是我的"

FeClaw 不宣传自己是"数字资产平台"——但要让用户在使用中自然感受到平台上的东西真正属于自己。

**怎么做（7 条原则）：**

① **开放格式，随时拿得走**
- 笔记导出 Markdown，录音导出 WAV/MP3，手写画布导出 PDF/SVG
- 不锁格式，不搞私有后缀。用户随时可以"把我的东西打包带走"

② **本地优先，云端是备份**
- Curio 原始记录层本地存储为主，云端是同步副本
- 没网也能打开自己的笔记、录音、手写画布。网络只是用来同步和让 AI 处理

③ **手写感 > 打字感**
- 你的笔迹、画布上的涂改痕迹、草稿的删改线条——全部保留
- 不是"打印体 + 整齐排版"——是你自己的笔迹原貌。翻出来看，感觉就是"这是我写的"

④ **不被平台绑架**
- FeClaw 倒了，你的笔记不会消失。本地有完整的一份
- 你不会因为没有订阅而读不了自己写的东西

⑤ **看得见的时间痕迹**
- 时间线视图 → 你看到"这是我上周写的"、"这是三个月前第一次做 PBL"
- 物理笔记本翻起来有厚薄、有新旧——数字版也能看到"时间的厚度"

⑥ **可分享、可赠送**
- 你写的一篇很好的知识总结，生成一个链接直接发给同学
- 同学点开看到的是你的笔迹、你的思路——不是 FeClaw 的广告

⑦ **"不存在"清理提示**
- 传统笔记软件会问"是否删除此笔记？"——用户会犹豫，"删了会不会找不回来？"
- Curio不做"删除"——做"归档"。归档 = 暂时看不见，但随时可找回
- 存储是你的设备的事，不是我们的事。你不需要帮我们省空间

**一句话：** 让用户感觉"这些东西是我的，只是暂时放在 FeClaw 帮我管着"——而不是"这是我租用 FeClaw 服务期间产生的，离开了 FeClaw 就没了"。
- **Universe** = 沉浸式 PBL 世界，你在里面学
- **Memory** = 你在真实世界捕获的所有学习痕迹，永久的个人知识库
- 两者可联动：你在 Universe 里学到的内容可以补充到 Memory；你在真实世界捕获的内容可以在 Universe 中被 Agent 引用

### 纸质数字化 Pipeline

```
拍照（手机/平板）
  ↓
OCR 结构化提取
  ├── 红笔标注 → 高亮提取 → 自动归集"你不知道的东西"
  ├── 草稿计算 → 错误路径识别 → "在哪步想歪的"
  ├── 试卷批注 → 错题归因（概念不清/计算错误/审题漏）
  └── 手写笔记 → 全文 OCR → 可检索文本
  ↓
入库到个人知识库
  ↓
AI 可用：
  - 提问："上次红笔标的电解池题，归纳一下规律"
  - 跨题关联："这跟上周犯的是一类错误"
  - 复习生成："根据红笔标注，帮我生成易错点清单"
```

### 信息收集
用户上网/翻书/拍照看到有趣的内容，无需立刻消化——只需随手捕获，AI 后台整理：
- 拍照路边三脚架设备 → AI 识别并解释："这是全站仪，用于工程测量的..."
- 收藏网页 → AI 提取核心结论 + 摘要
- 下次遇到相关内容 → 自动拉起："你之前拍过一张全站仪的照片..."

### 零通知设计原则（跨设备）
FeClaw 整体采取"零通知打扰"策略。Curio 捕获的内容跨设备自动同步，不推送任何通知。

- 手机捕获 → 自动进入 Curio 库 → Pad/Desktop 可见
- 不需要"从 A 传到 B"的概念——捕获即存在
- 除非用户在手机捕获时手动标记为"设置为待办"，否则不会产生任何提示
- 即使设置了待办，通知也是"静默式的"——只在回到主页时看到（如小红点或条目变化），不弹窗、不推送、不打断当前操作
- 核心原则：FeClaw 极其反对通知切割注意力。用户想处理了，自己打开 FeClaw 自然会看到

### 试卷答题设计（试卷区+草稿区）


#### 草稿区交互
草稿区设计为**浮动的半透明面板**——不占用试卷的屏幕空间，而是像"铺了一张草稿纸在试卷上"：

- **默认状态：** 试卷满屏。底部有一条细栏（约 20px），提示"🗒 向上拉起草稿纸"
- **展开状态：** 用手指从底部往上拉 → 草稿面板浮出
  - 背景：高斯模糊/云母效果（半透明），试卷内容透过模糊层隐约可见
  - 笔迹：半透明，在背景遮罩之上
  - 感受：不是分屏，是真的掏了一张草稿纸铺在试卷上
- **交互方式：** 笔写字，手指滚动
- **隐藏草稿：** 把面板向下推 → 自动收起到底部，回到试卷全屏
- **实际物理模拟：** 真实草稿纸是不透明的。此处为了视觉连贯性采用半透明——屏幕主体永远是试卷，草稿是"浮在上面的一张纸"

#### 草稿时间追踪
系统自动记录用户写草稿时的上下文：
- 当前在写哪一题（根据试卷区的操作位置推断）
- 草稿内容对应的时间戳
- 回顾时可以看到："T3 的草稿写在 19:15-19:18，花了 3 分钟，计算过程中有 2 次涂改"

### 灵感记录
随手记，极简，极快。借鉴成熟笔记产品的交互逻辑，AI 辅助补全 + 自动归类 + 关联提醒。

```
## 附录二：FeClaw Pad — 专属无纸化学习平板概念设计

### 定位
FeClaw Pad 不是推向市场的产品——是 FeClaw 创始人自己的私人学习平板。安装安卓平板，以 FeClaw 为默认启动器（Launcher），不装游戏/社交/娱乐 App，专用于无纸化学习。

### 开机主页（三态模型）

```
┌─────────────────────────────────┐
│   🖊 快速记录...                  │
│                                 │
│   📋 今天                        │
│   · 码头调研录音已整理 ✅         │
│   · 英语课笔记 → 待回顾          │
│                                 │
│   📌 Curio 注意到                │
│   · 录音里提到"阶跃 Step 模型"   │
│                                 │
│   📅 这周写了 7 篇笔记            │
└─────────────────────────────────┘
```

不显示时间、电池、Wi-Fi 图标——不制造无关注意力消耗。

### 三种模态

| 模态 | 触发方式 | 核心交互 |
|:----:|:--------|:--------|
| **书写态**（默认） | 从笔槽拿出笔 | 手写画布+5件工具栏 |
| **知识对话** | 画问号→长按 2s | Agent 自动查+答 |
| **调研态** | 点内容进入 | 录音/照片/整理结果 |

### 知识对话激活机制（统一手势：画问号 + 最后一笔长按）

#### 核心手势（通用）
**在任何可交互的内容上（手写画布、PDF、网页、照片）——**
- 用笔在需要提问的区域画一个普通问号"？"
- **最后一笔不抬笔，按住 2 秒再松开**
- → Agent 激活，开始解释

```
  [乙醇连续氧化]
   _______________
               ？  ← 画完整个问号后，最后一笔按住 2s
                  ↑
              普通画问号 = 不触发
              画完按住 2s = 触发搜索
```

**为什么是最后一笔长按：**
- 普通问号 = 你只是画了一个问号（可能是标点符号、可能是涂鸦）
- 长按问号 = 你认真想提问（激活 Agent）
- 同一手势，两种含义，物理上不会误触

#### 画布手写上的交互
在你写的手写笔记上：
1. 对一段文字画横线（选中区域）
2. 在旁边画问号
3. 最后一笔长按 2s → Agent 激活

横线 + 问号 + 长按 = 完整激活序列。缺一个都不触发。

#### PDF/网页/照片上的交互
在外部内容上：
1. 用笔圈出不懂的段落（画一个圆圈或下划线）
2. 在旁边画问号
3. 最后一笔长按 2s → Agent 激活
4. Curio OCR 提取圈选范围内的文字 → 传给 Agent

#### 追问规则
- 第一条问题：必须手写 + 长按（强制第一问经过思考）
- 后续追问：可用键盘（Agent 已在对话中）

**Agent 回复：** 以对话气泡出现在画布右侧。优先搜 Curio 知识库→联网→总结。用户看完可点"插入笔记"。

**回复的持久性：**
- **默认：系统默默记录问答历史** → Agent 后续可引用。"你上次问过乙醇氧化，现在跟你问的酯化反应有联系。"
- **可选不记录：** 回复卡片右上角有一个 🧹 按钮 → 点击该条问答记录不在任何地方保存。适合于隐私问题或临时无关查询。
- 当草稿纸展开时，主画布触发的画问号搜索会自动收起草稿纸——确保搜索弹窗背后是主画布内容，不是草稿纸。

### 扩展提问范围：PDF/文章划线提问

同样的问号长按机制不限于手写画布——可扩展到任何在 Pad 上查看的内容：

```
你在 Pad 上打开一篇 PDF 论文：
  读到一段关于注意力机制的描述——
  "multi-head attention projects Q, K, V h times"
  你不理解。

  你用笔在这段文字下面画横线 →
  画一个问号 → 长按 2s →

  Agent 解释：
  "多头注意力机制把 Q、K、V 分别投影 h 次，
   每次学习不同子空间的特征。"
```

**技术逻辑：** Curio 识别你画横线+问号的位置 → OCR 提取圈选范围内的文字 → 传给 Agent 解释。跟手写画布上的机制完全相同——只是标的物从你的手写字变成了 PDF 文字。

同样适用于：网页文章、老师发的课件截图、拍的白板照片。**任何你在 Pad 上看到的、不理解的内容——圈起来问就对了。**

### 刻意设计：围墙内搜索 = Agent 对话

FeClaw Pad 不设"浏览器"。手写时遇到不懂的，通过知识对话向 Agent 提问。Agent 负责过滤垃圾、呈现精华。90% 的知识查询问题（名词解释、公式、数据、考纲）可在围墙内解决，无需掏出手机。

### 刻意设计：键盘可用但不默认

书写态下默认无键盘。知识对话追问时可自动弹出键盘。但手写是第一交互——写下来的问题更经过思考，答案记得更牢。

### 与手机/Desktop 的关系

| 设备 | 用途 | 数据 |
|:----|:----|:----|
| 手机 | 灵感记录、拍照识别 | Curio 云端同步 |
| Pad | 手写笔记、深度整理 | Curio 本地优先 |
| Desktop | Agent 协作、大屏工作 | 同一份数据源 |

### 避免沦为"泡面盖"的考虑

- **生产型 > 消费型**：Pad 用于完成真实任务（PBL 报告/错题整理/调研分析），不是用于"读书"
- **Pad 是配件不是入口**：没有平板也能完整使用 FeClaw（手机/Desktop）
- **与 Curio 绑定**：Pad 吃灰了，你的知识资产在 Curio 里不会丢

### 待讨论问题

- 手写识别准确度与速度（中英文混写的挑战）
- 多设备同步冲突处理（手机拍 → Pad 批注 → Desktop 整理）
- 存储策略（~/Curio/ 目录结构的跨平台一致性）
- 用户数字资产所有权的实现细节

### Pad 功能可用性（含桌面模式）

| 功能 | 极简模式（默认） | 桌面模式（设置开启） |
|:----:|:--------------:|:-----------------:|
| Curio 手写画布 | ✅ 核心 | ✅ |
| 知识对话 | ✅ 核心 | ✅ |
| Agent 私教 | ✅ 浮窗 | ✅ |
| Universe | ❌ 默认隐藏 | ✅ 开启 |
| Work 群聊 | ⚠️ 只读 | ✅ 全功能 |
| 上帝模式 | ❌ | ✅ 移动备课 |
| 录音三件套 | ✅ | ✅ |

**桌面模式开关：** 设置中提供。关=FeClaw Pad 原始极简设计（专注优先）。开=体验对齐 Desktop，Pad 变触屏笔记本电脑（全功能可用）。用户可根据当前需求自由切换。外出备课的老师开桌面模式用上帝模式；自习的学生关桌面模式专注学习。

**跨屏推任务：** 不依赖桌面模式。手机用户可以把当前任务推到 Pad 上继续（拍试卷→"在Pad上继续"→Pad主页出现待办）。Curio 待办系统的跨设备版本。

**跨屏 URL 推送（外部内容入 Curio）：**
手机把 URL 推送到 Pad 时，不在目标设备打开浏览器——服务器端处理后转为 Curio 可读内容。

三级渲染架构：

| 层级 | 处理方式 | 交互能力 | 默认？ | 适用场景 |
|:----:|:--------|:--------:|:------:|:--------|
| T1 | LLM 强清洗（去广告/导航/评论/推荐，保留文字+图片） | 划线、批注、画问号 | ✅ 默认 | 90% 的文章/新闻/博客 |
| T2 | Playwright 截长图（保留完整视觉结构） | 只读不可交互 | ❌ 手动选 | 复杂排版/表格/多栏/代码高亮 |
| T3 | WebView 原生打开 | 完全可操作 | ❌ 用户强选+确认 | 必须交互的页面（表单/Demo） |

- T1/T2：画布内可读，不离开 Curio 上下文
- T3：弹出确认警告"可能分散注意力"，用户主动选择
- 所有层级：URL 处理完毕后自动存入 Curio，成为用户知识库的一部分

### 网课听讲模式（FeClaw 作为 Launcher）

#### 核心哲学：可以看但不能交互

FeClaw Pad 不内置视频播放器，但承认外部平台（B站/慕课）有优质课程资源。用户需要"看视频"但不应该跟视频平台的交互元素（推荐算法/评论区/搜索/首页）接触——因为一旦交互，注意力就被分散了。

**技术方案：MediaProjection 直播视图（仅 FeClaw 作为 Launcher 时可用）**
```
用户承诺"学生物第三讲"→ FeClaw 引导打开 B 站 →
  ① 系统弹出 MediaProjection 授权："FeClaw 要截取您的屏幕？" → 用户点"立即开始"
  ② FeClaw 后台截取 B 站画面 → 自动裁剪到视频实际画框（去掉周边按钮/推荐/评论区）
  ③ FeClaw 内显示纯视频直播画面（不显示 B 站任何 UI 元素）
  ④ 用户可操作：看视频、截图当前帧（自动贴入画布）、右侧拉出半透明画布记笔记
  ⑤ 无法操作：点击推荐视频、刷评论区、搜索、进首页、进 UP 主空间

  用户看到的不是"B 站 App"——是"从 B 站截取的视频画面"。
```

**权限要求（无需 root，无需系统签名）：**
- `MediaProjection` — 安卓标准截屏 API，用户需首次手动授权
- `SYSTEM_ALERT_WINDOW` — 悬浮窗权限，用户需手动在设置里打开

**挑战：DRM/版权保护内容**
- 部分付费/大会员视频设了 `FLAG_SECURE` → MediaProjection 捕获到纯黑屏
- 大学 MOOC/普通用户上传视频 — 大部分无 DRM，可正常捕获
- **容错：** 遇到黑屏 → 显示提示"该视频受版权保护，请回 B 站 App 观看" → 回退到手动截图+笔记方案
- 目标：覆盖 80% 日常学习场景，不承诺 100%

### 设计铁则：画布内禁止切换上下文

从画布页面除非回到主页，否则不打开任何新的独立窗口或页面。所有交互（搜索、聊天、查资料）必须是：
- **从下往上浮动的弹窗**
- **强制半透明**，背景透出画布内容
- 用户始终能看到自己原来在写什么——心理上锚定"当前主要任务"
- 关闭弹窗 → 回到画布的同一位置，光标不跳转，内容不变化

### 手写编辑：速度区分 + 自然手势

FeClaw Pad 的手写编辑不追求"系统猜你的意图"——而是用速度区分 + 有限手势集来保持简洁。

**删除手势（仅识别三种，其他划线=正常笔画）：**
- 快速在字中间画横线 —→ 删除
- 快速在字上画斜杠 / —→ 删除
- 快速在字上画反斜杠 \ —→ 删除
- 涂黑覆盖一个字的区域 —→ 删除

**速度区分：** 快速划过 = 编辑操作（删除/修改）。慢速画 = 正常笔画（线条/文字/图形）。用户第一次打开画布时显示一次性教学提示（画布角落灰色气泡，3 秒自动消失）。

**安全网：** 任何误删除操作，1 秒内页面底部浮现"撤回"按钮。

**拼音/生字输入：**
写"lian" → 划线 → 画问号 → 长按 → 系统检测可能是拼音 → 弹候选菜单：连/脸/练/莲 + [搜索] 按钮。点候选 → 不直接插入正文——在划线位置旁边出现一个 (!) 参考标记。看到字符后自己拿笔写出完整词。写完点 (!) 标记，它消失。参考标记中的印刷体始终不进入手写正文。

**英语查词：**
写"book" → 划线 → 画问号 → 长按 → 弹候选：英（[英] n.书 v.预订）+ 中文（嫦娥）→ 点选后同样以 (!) 参考标记形式出现。

**编辑后整理：** 写完可点"生成整理版"→ Agent 将所有修改标记应用到正文 → 输出纯文本/Word/格式文本。原始手写笔迹始终保留。

### 底层架构原则：Pad = 记录终端，云端 = 计算引擎

所有"理解"类操作统一走 VLM。Pad 上不运行任何本地模型。

| 操作 | 本地做什么 | 云端做什么 |
|:----|:---------|:---------|

### Curio（格物所）在全局架构中的定位

Curio 不是"经典 FeClaw Agent 系统的一部分"——它是与 Agent 对话层**平级的独立层**，互相可调用但不互相依赖。IPO 模型中：Curio = Input（及少量 Process），Agent = Process & Output。

**Agent 的三层输入（"是什么"层，不重叠）：**
1. **Curio（个人知识）** → 人类亲自产出的内容（手写笔记/拍照/录音/灵感速记/问答记录/URL清洗结果）。走心。
2. **knowledge_search（公共知识）** → 教材/考纲/公开趋势索引。FeClaw 统一维护，所有 Agent 共享。走教材。
3. **web_search（互联网）** → 实时公共信息。走网络。

**Curio vs idx-{hash}-kb（VFS自动向量化）：**
- Curio 内容 = 你在 FeClaw 上的行为副产品（你亲自写了/记了/拍了/问了）
- VFS/kb 内容 = 你丢进工作区的文件被 AI 消化后的影子（AI 替你读的）
- 两者互补不同质。对 Agent 都是"读一下"。
```
