## 1. 一句话定位

**FeClaw = 学习的第一入口。**

一个高中生打开手机/平板/电脑，第一个想到的不是微信聊天、不是抖音刷视频——是打开 FeClaw，因为那里有惦记的 Agent、有写到一半的 Curio 笔记、有一个等着你探索的宇宙。

---

## 2. 用户是谁

**一个 14-18 岁的高中生。**

- 在中国读高一或高二，选科了（物化生/历政地）
- 每天在学校上课、写作业、考试
- 手机被允许带进学校但上课不能用
- 可能有一台平板在家，也可能没有
- 70% 的用户只有手机
- 不在 FeClaw 上刷题（那是作业帮的事），FeClaw 帮他管「学过的东西」

## 3. 产品长什么样

### 3.1 手机端

底部两个 Tab：

| Tab | 内容 |
|:----|:------|
| **💬 聊天** | Agent/Work/Universe 混排对话列表（用户不区分底层引擎） |
| **📦 Curio** | 见 `02-curio.md`——个人知识库 |

**聊天 Tab** 是传统 IM 样式，对话列表按时间倒序，点进去 Agent 聊天界面。桌面端保持现有的三栏布局（Agent 列表 + 对话 + 侧栏）不变。

**Curio Tab** 是 Flomo 式极简主页 + 时间线 + 画布。

### 3.2 Pad 端

| 模式 | 默认？ | 说明 |
|:-----|:-----:|:------|
| **Curio 画布** | ✅ 默认 | 手写/录音/拍照三件套，Curio 主场 |
| **桌面模式** | 🔘 可选 | 对齐 Desktop 全功能（Universe/Work/God Mode），类似安卓 WPS 桌面版 |

手机和 Pad 共享同一份 Curio 数据——手机拍的照片同步到 Pad 画布，Pad 写的手写笔记在手机时间线上可见。

### 3.3 Desktop 端

保持现有三栏 IM 布局不变。Curio 通过 Web 浏览器使用。本地模式可离线运行。

## 4. 核心使用场景

### 场景 A：Agent 私教

用户在聊天界面与 Agent 对话：
- 问一道物理题 → Agent 搜教材知识库 + 深度推理
- 拍一张试卷 → Agent 识别错题、归因、整理
- 英语作文批改 → Agent 全文批注
- 所有产生的内容，用户可主动存入 Curio

### 场景 B：Curio 沉淀

用户日常学习的碎片汇聚到 Curio：
- **拍照：** 试卷/笔记/板书 → VLM 结构化入库（印刷体→HTML，手写→二值化净图）
- **录音：** 讲座/课堂/采访 → ASR 转写 + 关键提取 + 画布手写 + 照片时间锚定
- **速记：** flomo 灵感输入 → AI 补全 + 自动分类
- **链接：** 网页/文章 → LLM 清洗去脏，只留内容

### 场景 C：Universe 沉浸

用户进入一个虚拟世界，跟角色互动学东西：
- 枫叶镇（英语）：跟 Kevin 去 cafeteria 点单
- 澜汐小镇（理科）：帮钟梓琪处理潮汐数据

（详见 `03-universe.md`，V2 目标）

### 场景 D：Work 协作

多 Agent + 真人同群协作（详见 `04-engine.md`）。
