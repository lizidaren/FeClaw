## 1. 一句话定位

**FeClaw = 学习的第一入口。**

用户打开手机/平板/电脑，第一个想到的不是打开微信聊天、不是抖音刷视频——是打开 FeClaw，因为那里有惦记的 Agent、有写到一半的 Curio 笔记、有一个等着你探索的宇宙。

---

## 2. 困境：AI 学习产品为什么还不够好

### 2.1 交易型产品 vs 关系型产品

当前主流 AI 学习产品（Kimi、豆包、DeepSeek、ChatGPT）是 **问答型** 的——你问它答，交易结束。用户是"产品经理"视角：回答不好就改 prompt 重新回车。这偏离了学习本身。

FeClaw 是 **关系型** 的——用户遇见一群有记忆、有人设、有关系的 Agent，学习是参与这个世界的副产品。

### 2.2 用户数字资产的碎片化

一个高中生的学习数据分布在：

| 渠道 | 数据 | 问题 |
|:----|:-----|:-----|
| 微信 | 聊天记录、PDF、照片 | 不可检索、不可整理 |
| 拍照 | 试卷、笔记截图 | 躺在相册吃灰 |
| B站 | 网课录屏 | 看完就忘 |
| 纸质笔记本 | 手写笔记 | 不能搜索、不能同步 |
| 作文本 | 老师批改 | 改完就丢 |

FeClaw 的 Curio（格物所）把这些全部汇入一条时间线——拍照、录音、灵感速记、网页剪藏，都在一起。可检索、可回顾、可关联。

## 3. 产品哲学：三条红线

### 3.1 Curio 域（学习专注）

| # | 原则 | 含义 | 典型设计 |
|:-:|:----|:------|:--------|
| 1 | **不切割注意力** | 无弹窗通知、无推送、画布内不切换上下文 | 主页静默横幅、半透明浮窗不离画布、零推送 |
| 2 | **不强迫** | 无分类要求、无必填项、不做模式学习 | flomo 式输入、纯时间线不设文件夹 |
| 3 | **画布为王** | 手写是默认交互、笔+手指分工明确 | Pad 默认书写态，笔写文字/手指滚动/手势编辑 |
| 4 | **数据所有权** | 开放格式、本地优先、所有操作可逆 | ~/Curio/ 目录直接存文件、原始层不可变、没有删除只有归档 |
| 5 | **零通知** | 不在系统通知栏推送，只在主页可见 | 完成度在主页显示，跨屏任务不响铃不弹窗 |
| 6 | **完成度追踪** | 有始有终，帮你记住做到哪 | 画布关闭自动标记"未完成"、24h 降级 |
| 7 | **克制边界** | 不浏览器、不社交、不游戏 | 围墙搜索（问 Agent 代替搜网页） |
| 8 | **可以看但不能交互** | 外部内容只读化，杜绝被算法带走 | T1/T2 URL 清洗只显示纯内容 |
| 9 | **Curio 六原则** | 输入即整理、不整理>整理、回顾>记录、克制边界、不评价、可信主动 | flomo 超集、纯时间线 |
| 10 | **防信息过载** | 所有输入通道统一清洗，去脏留净 | 网页→LLM 去广告、PPT→VLM 去摩尔纹 |

### 3.2 用户数据所有权（7 条铁则）

1. **开放格式** — Markdown / SVG / MP3 / PNG，不使用专有二进制格式
2. **本地优先** — Curio 主数据在本地，云端只是同步副本。离线=常态，在线=同步
3. **手写笔迹保留** — AI 产物在独立层，不覆盖原始笔迹
4. **无平台锁定** — 一键导出所有笔记为通用格式
5. **时间深度** — 每条笔记保留创建/编辑/回顾/关联的时间戳
6. **可分享** — 分享链接无需对方有 FeClaw 账号
7. **没有删除只有归档** — 标记"删除"实际归档到 `.trash/`，30 天内可恢复

### 3.3 技术红线：All LLM, No NLP

> 2026-07-13 明确记录。

FeClaw 是 AI-native 产品，所有判断尽可能走 LLM，不引入传统 NLP（词频统计、正则匹配、语言学规则等）。

| ✅ 允许 | ❌ 不允许 |
|:--------|:---------|
| LLM 判断语言倾向 | 词频统计检测语言 |
| LLM 判断内容相关性 | TF-IDF / BM25 关键词匹配 |
| LLM 判断对话是否走题 | 余弦相似度阈值切割 |
| LLM 驱动二值化阈值调优 | 自适应阈值算法（Otsu 等）|

**例外：** keyword 类型（关键词硬匹配）可用于少数明确场景（如用户提及"重置"等系统敏感词），但需标注假阳性风险。

## 4. 双基座架构

> 2026-07-13 决策更新：FeClaw 由 Agent Harness 和信息化生产工具两个基座构成，而非单基座。

### 4.1 架构总览

```
┌──────────────────────────────────────────────┐
│                产品层                         │
│ Agent │ Curio │ Work │ Universe              │
├──────────────────────────────────────────────┤
│            双基座                              │
│ Agent Harness       │ 信息化生产工具基座        │
│ (LLM/VFS/搜索/路由) │ (Univer Sheet/Doc/Slide │
│                      │  + PDF + 导入/导出)     │
└──────────────────────────────────────────────┘
```

### 4.2 Agent Harness（基座）

Agent Harness 是 FeClaw 的智能层，提供一切 AI 底层能力：
- **Agent 聊天** — 1v1 私教、群聊协作
- **智能路由（SmartRouter）** — 小模型预分类 + 大模型深度回答
- **知识检索** — 教材向量库 + 联网搜索 + Curio 个人库
- **工具系统** — 文件读写/代码执行/网络爬取/子 Agent
- **多渠道统一** — Web / WeChat / Desktop WS / Mobile API 共用一个 Agent 大脑

### 4.2 信息化生产工具基座

信息化生产工具是 FeClaw 的生产力层，提供一切文档/数据处理能力：

- **文档能力** — 嵌入 Univer Sheet/Doc/Slide 进行查看和编辑
- **PDF 渲染** — 基于 pdf.js 实现 PDF 查看和标注
- **导入/导出** — 统一文档入库管线，支持 Word/Excel/PPT/PDF/图片
- **模板系统** — 支持报告模板、数据分析模板
- **与 Agent 联动** — Agent 可直接操作文档内容（分析表格、写作润色、生成演示文稿）

> 信息化生产工具是水平层，不属于任何一个产品模块——Agent、Curio、Work、Universe 都可以调用它。选型参考：Univer（开源，TypeScript，Apache-2.0，插件化架构）。

### 4.3 Curio（格物所 — 桥梁层）

Curio 连接"现在的学习"和"未来的 PBL"：

- 用户今天用 Agent 辅导学习，产生笔记/录音/拍照 → 自动入库 Curio
- 明天进入 PBL 宇宙时，Curio 里的积累成为个人上下文
- 没有 Curio，PBL 宇宙里的学生就没有历史、没有积累

### 4.3 PBL Universe（未来教育）

Universe 是 FeClaw 的差异化天花板：
- **沉浸式虚拟世界** — 全英文的枫叶镇、理科的澜汐小镇
- **NPC Agent** — 有记忆、有人设的固定角色（Kevin / 钟梓琪）
- **种子系统** — 教师设计世界观，AI 自动演化
- **Work 协作** — Agent 进入真实项目群，作为协作者

## 6. 四大使用场景

### 6.1 Agent 私教（Harness）

用户在聊天界面中与 Agent 对话：

- 问一道物理题 → Agent 搜教材知识库 + 深度推理
- 拍一张试卷 → Agent 识别错题、归因、整理
- 英语作文批改 → Agent 全文批注

### 6.2 Curio 沉淀

用户日常学习中的捕获汇聚到 Curio：

- 拍照：试卷/笔记/板书 → VLM 结构化入库
- 录音：讲座/课堂/灵感 → ASR 转写 + 关键提取
- 速记：flomo 灵感输入 → AI 补全 + 分类
- 链接：网页/文章 → LLM 清洗去脏只留内容

### 6.3 Universe 沉浸

用户进入一个虚拟世界，跟角色互动：

- 枫叶镇（英语）：跟 Kevin 去 cafeteria 点单、跟 Sarah 讨论美国文学
- 澜汐小镇（理科）：帮钟梓琪处理潮汐数据、向陈晓棠挑战数学题
- 学习发生在"我想喝咖啡所以学会了用英语点单"的过程中

### 6.4 Work 协作

- 多 Agent + 真人同群：AI PM 分配任务、AI 程序员写代码、真人审核
- 群共享空间：共享 VFS 存储
- 群广场：动态发布

## 7. 渠道体系

FeClaw 的渠道不是"外部集成"，而是平等的通信入口：

```
Agent 大脑 (ChatHistory + 统一上下文)
  │
  ├── Web（Jinja2 SSR → 渐进 SPA）     — 浏览器入口
  ├── WeChat（iLink 协议）              — 微信入口
  ├── Desktop WS（Tauri WS 隧道）       — Windows 原生入口
  └── Mobile API（REST + WS）           — 移动端入口
```

所有渠道共享同一个上下文——Agent 不知道你是从 Web 还是微信还是 Desktop 发来的消息。它只看到一条新的 ChatHistory 记录。

## 8. 竞品映射

| 产品 | 形式 | FeClaw 差异点 |
|:----|:-----|:-------------|
| 豆包 / Kimi /  DeepSeek | 问答型 AI | **关系型**：有记忆、有人设的 Agent |
| GoodNotes / Notability | 笔记软件 | **注意力十年**：防过载洗净、不强迫分类 |
| 荣耀伴学知识库 | 系统级 AI 笔记 | **跨平台+开放格式+数据所有权** |
| Duolingo |  gamified 学习 | **全景式 PBL**：不是碎片化练习 |
| Socratopia | 角色导师 | **Universe 世界**：不是 1v1 聊天 |
| Flomo | 笔记 | **Curio 超集**：画布+录音+拍照+@引用+完成度 |

FeClaw 的独特定位：
- 开源 + 开放格式（数据不锁死）
- 跨平台同步（不绑定硬件）
- Agent 人格 + PBL Universe（学习体验非工具化）

## 9. 合规边界

FeClaw 的产品定位避开《人工智能拟人化互动服务管理暂行办法》（2026-07-15 生效）的核心靶区：

- ❌ 不是 AI 情感伴侣
- ❌ 不涉及持续的、拟人化的情感互动
- ✅ 自定性倾向：学习教育 / 智能客服 / 知识问答 / 工作助手

**注意事项：**
- Universe 中若出现长期 NPC 关系，对话设计需避免情感依赖
- 用户协议中明确说明
- 未成年人模式下限制特定类型对话
